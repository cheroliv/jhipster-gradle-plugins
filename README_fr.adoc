Mono-dépôt contenant deux plugins Gradle pour les projets JHipster Kotlin.
Chaque plugin est un artefact indépendant publié séparément sur le Gradle Plugin Portal.

== Vision

Les projets JHipster subissent une tension fondamentale : le générateur produit
une base de code complète, mais chaque régénération écrase tout — y compris
le code métier écrit à la main.
Ce dépôt résout ce problème avec deux plugins complémentaires qui couvrent
l'ensemble du cycle de vie d'un projet JHipster Kotlin.

image::/.github/workflows/readmes/images/fr/vision-overview.png[vision-overview]

== Plugins

=== `dev.jhipster.persistence`  ✅ stable

Orchestre le cycle clean → generate → sync d'un projet JHipster
sans perdre le code métier Kotlin persisté dans `__codebase__/`.

[source,kotlin]
----
// workspace/build.gradle.kts
plugins {
    id("dev.jhipster.persistence") version "1.0.0"
}
----

> Aucune configuration DSL requise — tout résolu par convention.

=== `dev.jhipster.assistant`  🚧 SNAPSHOT

Assistance au code via RAG pgvector, LLM multi-provider et serveur MCP.
Non publié — développement démarrera après stabilisation de `persistence`.

== Structure du dépôt

image::/.github/workflows/readmes/images/fr/repo-structure.png[repo-structure]

[source]
----
jhipster-gradle-plugins/
├── settings.gradle.kts          ← include("persistence", "assistant")
├── build.gradle.kts             ← conventions communes (toolchain, test, publish)
├── gradle/
│   └── libs.versions.toml       ← catalog partagé, versions séparées par plugin
├── persistence/                 ← dev.jhipster.persistence (léger)
│   ├── build.gradle.kts
│   └── src/
└── assistant/                   ← dev.jhipster.assistant (SNAPSHOT)
    ├── build.gradle.kts
    └── src/
----

== Séparation des dépendances

La frontière de module Gradle garantit physiquement que `persistence`
ne peut pas tirer une dépendance LangChain4j ou Docker accidentellement.

image::/.github/workflows/readmes/images/fr/dependency-boundary.png[dependency-boundary]

== Commandes racine

[source,bash]
----
# Compiler et tester les deux modules
./gradlew check

# Persistence uniquement
./gradlew :persistence:check

# Publier persistence sur le Gradle Plugin Portal
./gradlew :persistence:publishPlugins

# Publication locale (dépôt Maven dans build/local-repo)
./gradlew :persistence:publishToMavenLocal

# Assistant uniquement (SNAPSHOT, non publié)
./gradlew :assistant:check
----

== Prérequis

[cols="2,3", options="header"]
|===
| Prérequis | Détail

| JDK 24+
| Toolchain Gradle configurée automatiquement

| nvm + Node 22
| Alias `lts/jod` requis par `generateJdl`

| JHipster CLI
| `npm install -g generator-jhipster`

| Docker + Compose v2
| Pour les tâches `dockerDb`, `dockerUp`, `dev`

| rsync (optionnel)
| Fallback Java natif si absent
|===
