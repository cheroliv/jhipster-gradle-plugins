Mono-repository containing two Gradle plugins for JHipster Kotlin projects.
Each plugin is an independent artifact published separately on the Gradle Plugin Portal.

== Vision

JHipster projects face a fundamental tension: the generator produces
a complete codebase, but each regeneration overwrites everything — including
hand-written business code.
This repository solves that problem with two complementary plugins covering
the entire lifecycle of a JHipster Kotlin project.

== Plugins

=== `com.cheroliv.jhipster.persistence`  ✅ stable

Orchestrates the clean → generate → sync cycle of a JHipster project
without losing the Kotlin business code persisted in `__codebase__/`.

[source,kotlin]
----
// workspace/build.gradle.kts
plugins {
    id("com.cheroliv.jhipster.persistence") version "0.0.0"
}
----

> No DSL configuration required — everything resolved by convention.

=== `com.cheroliv.jhipster.assistant`  🚧 SNAPSHOT

Code assistance via RAG pgvector, multi-provider LLM and MCP server.
Not published — development will start after `persistence` is stable.

== Dependency Boundary

The Gradle module boundary physically guarantees that `persistence`
cannot accidentally pull in a LangChain4j or Docker dependency.

[plantuml, dependency-boundary, svg]
----

  Resolution time: > 30s

== Root Commands

[source,bash]
----
# Build and test both modules
./gradlew check

# Persistence only
./gradlew :persistence:check

# Publish persistence to the Gradle Plugin Portal
./gradlew :persistence:publishPlugins

# Local publication (Maven repository in build/local-repo)
./gradlew :persistence:publishToMavenLocal

# Assistant only (SNAPSHOT, not published)
./gradlew :assistant:check
----

== Requirements

[cols="2,3", options="header"]
|===
| Requirement | Details

| JDK 25+
| Gradle toolchain configured automatically

| nvm + Node 22
| Alias `lts/jod` required by `generateJdl`

| JHipster CLI
| `npm install -g generator-jhipster`

| Docker + Compose v2
| Required for `dockerDb`, `dockerUp`, `dev`

| rsync (optional)
| Native Java fallback if absent
|===
