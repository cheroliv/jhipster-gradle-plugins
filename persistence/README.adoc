Gradle plugin that solves the fundamental problem of the JHipster cycle:
regenerating the project from a JDL file without losing the Kotlin business code.

== The Problem

JHipster is a full-stack Spring Boot application generator.
Its power lies in regeneration from a `.jdl` file —
but each regeneration overwrites the entire project,
including hand-written business code.

[plantuml, problem-statement, svg]
....

== The Solution: `__codebase__/`

The plugin introduces a sanctuary directory `__codebase__/` at the root of
the JHipster project. All Kotlin business code is persisted there.
After each regeneration, the plugin rsyncs `__codebase__/` back into the
regenerated project — within seconds, the project is complete again.

== Installation

=== System Requirements

[cols="2,3", options="header"]
|===
| Requirement | Details

| JDK 25+
| Toolchain automatically configured by the plugin

| nvm
| Installed in `~/.nvm` with alias `lts/jod` (Node 22)

| JHipster CLI
| `npm install -g generator-jhipster`

| Docker + Compose v2
| Required for `dockerDb`, `dockerUp`, `dev`

| rsync
| Optional — native Java fallback if absent (cross-platform)
|===

The plugin locates the JHipster project by *strict convention*,
with no explicit configuration.

  #D5F5E3 :Use this directory\nas the JHipster target project;
else (❌ no)
  if (A sibling contains\n**build.gradle(.kts)**?) then (✅ yes)
    #FEF9E7 :Use this directory\n(fallback without .yo-rc.json);
  else (❌ no)
    #FADBD8 :Explicit error —\nno JHipster project found;
    stop
  endif
endif

:Instantiate **ProjectConventions**\nwith the resolved directory;

:Register tasks\nfor group **jhipster-persistence**;

stop
@enduml

[cols="2,3", options="header"]
|===

| `jdlFile`
| `<project>/<project-name>.jdl`

| `dockerDir`
| `<project>/src/main/docker/`

| `dockerDbFile`

=== Task Reference

[cols="2,1,3", options="header"]
|===
| Task | Group | Description

| `regenerate`
| `jhipster-persistence`
| Full pipeline: `cleanCodebase` → `generateJdl` → `syncCodebase`

| `cleanCodebase`
| `jhipster-persistence`
| Removes JHipster-generated directories and files (`.gradle/`, `build/`, `src/`, `node_modules/`, etc.)

| `generateJdl`
| `jhipster-persistence`
| Runs `jhipster jdl <name>.jdl --force` via nvm `lts/jod`. Applies post-generation Kotlin patches.

| `syncCodebase`
| `jhipster-persistence`
| Rsyncs `__codebase__/` → regenerated project (`build.gradle`, `gradle/`, `src/`, `buildSrc/`, `.github/`). Native Java fallback if rsync is absent.

| `dockerDb`
| `jhipster-persistence`
| `docker compose up -d` on `postgresql.yml` (convention)

| `dockerUp`
| `jhipster-persistence`
| `docker compose up -d` full stack

| `dockerDown`
| `jhipster-persistence`
| `docker compose down`

| `dev`
| `jhipster-persistence`
| `dockerDb` then `./gradlew bootRun` in the JHipster target project
|===

== Common Usage

=== Modifying the Data Model

[source,bash]
----
# 1. Edit the JDL file
vim edster/edster.jdl

# 2. Regenerate (clean -> generate -> sync)
./gradlew regenerate

# 3. Start in development mode
./gradlew dev
----

=== Sync Only (Without Regenerating)

[source,bash]
----
# After manually modifying __codebase__/
./gradlew syncCodebase
----

=== Managing Docker Separately

[source,bash]
----
# Start PostgreSQL only
./gradlew dockerDb

# Start the full stack (PostgreSQL + Elasticsearch + etc.)
./gradlew dockerUp

# Stop all services
./gradlew dockerDown
----

== Sync Manifest (`SyncCodebaseTask`)

Artifacts synchronized from `__codebase__/` to the JHipster project:

[cols="2,2,1", options="header"]
|===
| Source in `__codebase__/` | Destination in project | Type

| `build.gradle`
| `./build.gradle`
| File

| `gradle/`
| `./gradle/`
| Directory

| `src/`
| `./src/`
| Directory

| `.github/`
| `./.github/`
| Directory

| `buildSrc/`
| `./buildSrc/`
| Directory
|===

TIP: `build/` and `.gradle/` are cleaned before synchronization
to avoid Gradle cache conflicts.

== Post-Generation Patches (`GenerateJdlTask`)

After each JHipster regeneration, these patches are applied automatically:

[cols="2,3", options="header"]
|===
| Patched File | Change

| `.gitignore`
| Adds `.goose`, `README.pdf`, `README.html`, `README.docx`, `README.epub` (fr and en)

| `gradle/libs.versions.toml`
| Replaces `kotlin = "x.y.z"` with `kotlin = "2.3.20"`

| `build.gradle`
| Injects `kotlin.jvmToolchain(25)` (idempotent via comment marker)

| `buildSrc/build.gradle`
| Updates `kotlin-gradle-plugin:x.y.z` to `2.3.20`
|===

== Common Error Reference

[cols="2,3", options="header"]
|===
| Error | Solution

| `No JHipster project found`
| Verify that the sibling project contains `.yo-rc.json`

| `JDL file not found`
| The JDL file must be named `<project-name>.jdl`

| `nvm required`
| Install nvm and create the alias `nvm alias lts/jod lts/jod`

| `jhipster: command not found`
| Run `npm install -g generator-jhipster` with nvm `lts/jod` active

| `__codebase__ not found`
| Create the directory manually or run `syncCodebase` after `generateJdl`
|===
